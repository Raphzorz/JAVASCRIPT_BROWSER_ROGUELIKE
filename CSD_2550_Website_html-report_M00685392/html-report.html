The main semantic element was used to signify content that is unique to the specific page or document. The main encompasses a large amount of content. What CSS formattings are used and where are they used?

Very many CSS formattings were used throughout the document. Some of the more commonly used ones are listed below

Padding and Margins

Paddings were applied around elements so as to create space between one element and another. Padding was applied to nearly every element seen inside of the webpage. Margins were also used to create space between one element and another however this is created using space outside of the element borders while padding creates space inside of its borders. The text "Copyright Chrono Spark 2018" has a margin-top applied to it so as to create space between itself and the social icons above it. On the other hand the hamburger icon that appears on smaller screens uses "padding: 0.75rem;" to manipulate its size.

Background

The background property was used to add images to the background. This can be seen in all screenshots as the main background. The background property has many other properties that can be used to manipulate it such as "background-repeat" which was set as "no-repeat" to ensure the background is not tiled, "background-attachment" which was set as fixed so as to ensure nothing changes on scroll, and background "size" which was set to "cover" so as to ensure that the background always covers the whole screen. A linear gradient was also applied to the background using the "linear-gradient" property

Shadows and Borders

Shadows and borders were applied to images so as to make them look less "flat" and more 3d. Border-radius was used to add rounding to some image corners. Both of these aspects can be seen in the images in Home page screen shot 2 as well as in the play-guide screen shots.

Font Properties

Multiple font properties were used throughout the document. Different sections of the document have different fonts, different font sizes, different colors, different font weightings(or boldness) and different line spacing. The main "Chrono-Trigger" text in the first screenshot can be seen to be vastly different to the text in the second screen shot. This difference was achieved through the aforementioned properties

Color

The concept of color was used throughout the document.It was applied to different texts, headings and backgrounds. Gradients were also used to enhance the overall feel of the website.

Animations

A few animations were used to make the website less "static". The initial text on the front page vibrates slightly while an animation is also played whenever a user mouses over the "sign-up" button. When a user mouses over images inside of the play-guide the image background changes, the text also changes and the overlay encapsulates the whole of the grid item. In the sign-up page the png image of the oracle slowly floats up and down. This effect was achieved by setting up a simple loop where the image is translated up and down over a period of 2 seconds

Height and Width

The height and width attributes were often used to set the maximum or minimum dimensions of elements. They were applied to most elements seen inside of the webpage

Flexbox and Grid

Flexbox and Grid were both used to position elements in a structured format. The play-guide page uses grid to order the different images in a symmetrical grid fashion. The grid items themselves are then manipulated flex-box to position the text overlay

What are classes and ID's and where are they used? Why are they needed?

Classes

Classes are applied to content elements inside of the HTML markup. These classes are then used inside of the CSS file to point to particular elements and style them in some way. Usage of classes allows elements of the same type to be styled differently. For example instead of having to have all h1's as the same style the class "landing" is given to the h1 inside of the index page. This is then styled inside of the CSS file in a different way to the h1 without the class "landing"

Classes are used throughout the document wherever content needs to be styled differently to other elements of the same style. Clasees are also used to avoid duplicating code for elements which you do need to share common style attributes. Multiple classes are applied to some elements to inherit different styles defined in different classes

The semantic time tag was used whenever a date or time was defined. In the code example the "PT" refers to duration

IDs

IDs work in the same way as classes, however the same id can only be defined once per document. It must be unique within that HTML document. IDs are primarily used by javascript but may be used in a CSS stylesheet in the same way as classes apart from the fact that a "#" must be used instead of a "." prior to selecting the ID. IDs have the advantage of having the ability to be referenced by hyperlinks through the use of the "href=" code. IDs were used inside of the navBar and the sign-up page so as to allow for simple javascript functions to act on the elements.

Grid Layout: CSS Grid And Bootstrap Grid

Bootstrap Grid

Bootstrap grid was used across multiple pages. It uses a combination of containers, rows and columns to structure the page. A grid in bootstrap is split into 12 pre-defined columns and multiple rows. Homepage screen shot 2 shows one such example. The code snippet shows how the div in which the images are located is given the class "col-md-4". The col refers to columns, the "md" means that the structure will apply to medium, large and extra large devices but not small. or extra small. The 4 refers to taking up 4 "slots" of the 12. As 3 of these columns are generated it means that every element will take up an equal 1/3rd of the allocated width. Home page screen shot 3 shows a different layout. Two columns are created using "col-lg-6". One column contains the text while the other contains the image. Both take up half the available width due to 12/6

CSS grid

CSS grid was used inside of the play-guide to set up a responsive grid structure. Inside of the CSS a "template area" was defined. The grid is made up of 6 columns and 5 rows on a large display. The grid gap is responsible for creating the padding between grid items. Once the "template" has been defined inside of the CSS the grid items are assigned to every template area ( as can be seen inside of the code snippet). This leads rise to the play-guide page structure. Different templates are set up for different screen widths so as to make the page mobile responsive.
